# VTEX IO Custom App SCSS Structure

## When to use this skill
Use this skill when the user is developing a custom VTEX IO app and needs to:
1. Apply styles to a specific block using the `blockClass` property.
2. Organize SCSS files within the `src/store/blocks` directory structure.
3. Write CSS selectors that correctly target the generated Handlebars/React container.
4. Prevent style leakage by correctly scoping rules.

## The Pattern: blockClass and Containers
VTEX IO custom apps often use a specific convention where the CSS class defined in the JSON block is transformed into a modifier class on the HTML container.

### 1. JSON Configuration (The Trigger)
First, the block must have a `blockClass` defined in its props.
*   **Format:** String or Array of strings.
*   **Naming Convention:** Short, lowercase, kebab-case (no spaces).

```json
{
  "container#example": {
    "children": ["custom-list#example-list"],
    "props": {
      "blockClass": "example-container"
    }
  },
  "custom-list#example-list": {
    "props": {
      "blockClass": "example-list"
    }
  }
}
2. File Location (Directory Structure)
Do not place these styles in a generic styles.css at the root. You must locate the SCSS file relative to the block definition to ensure the builder collects it.
Path Pattern: src/store/blocks/<Context>/styles/<FileName>.scss
The builder automatically concatenates all *.scss files found in src/store/blocks/**/styles/ during the build process.
3. SCSS Selector Convention (The Rule)
To target the element, you must use the modifier class prefix: .container--.
Syntax: .container--{blockClass}
Example: If the JSON has "blockClass": "example-list", the SCSS must start with:
/* src/store/blocks/home/styles/example-list.scss */

/* Target the outer container generated by the system */
.container--example-list {
  width: 100%;
  display: flex;
  flex-direction: column;

  /* Use nested selectors for internal elements rendered by your app */
  /* These class names should be defined in your React component */
  .imagesWrapper {
    width: 120px;
    height: 120px;
    overflow: hidden;

    .imageElement {
      width: 100%;
      object-fit: cover;
    }
  }

  .text {
    margin-top: 0.25rem;
    text-align: center;
  }
}
Best Practices and Constraints
Scope and Isolation
• Always nest styles inside the .container--{blockClass} selector.
• Never write root-level selectors (e.g., just .text { ... }) in these files, as they will leak to the entire store.
Responsiveness and Variables
• Use project variables (SASS variables) for colors and spacing instead of hardcoding values.
• Use utility mixins for media queries if available in the project architecture (e.g., @include media-max(...)).
File Organization
• Keep one SCSS file per block or variant.
• Filename should match the purpose or the blockClass for easier maintenance.
Workflow Summary for the Agent
1. Ask the user for the blockClass name defined in the JSON.
2. Generate the SCSS file content starting strictly with .container--{blockClass}.
3. Instruct the user to save the file in the src/store/blocks/.../styles/ folder.
4. Remind the user to run vtex link to trigger the build process that concatenates the styles.
