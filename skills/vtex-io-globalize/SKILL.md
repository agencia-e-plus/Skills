# VTEX IO Globalize & Advanced Styling

## When to use this skill
Use this skill ONLY when standard CSS Handles cannot solve the problem, specifically for:
1. Styling "inner HTML" content (e.g., `rich-text` blocks where you cannot add handles to internal tags).
2. Styling elements generated by third-party libraries (e.g., sliders, widgets) that inject their own DOM.
3. Using complex CSS selectors forbidden in standard files (e.g., `>`, `+`, `~`, `:has()`).
4. Creating global CSS resets or normalizations.

## Critical Technical Concept
In VTEX IO, component CSS is sandboxed. However, the `vtex.login` app is an exception. The **Globalize** builder feature acts as a transporter: it cuts specific CSS rules from your component's styling and appends them to `vtex.login.css` during the build process. This bypasses the standard sanitization but makes the styles globally accessible.

## Implementation Methods

### Method 1: The SCSS Mixin (Recommended)
This is the safest way to implement globalize, abstracting the syntax to prevent errors.

**1. Define the Mixin**
Add this to your global tools or mixins file:
```scss
@mixin globalize() {
  :iobuilderglobal {
    @content;
  }
}
2. Apply in Component SCSS Always nest the mixin inside a scoped class (handle) to prevent the styles from leaking to the entire store unintentionally.
/* vtex.store-theme.scss */

.wrapperContainer {
  /* Standard styles remain here */
  background-color: #fff;

  /* Globalized styles are transported to vtex.login.css */
  @include globalize {
    /* You can now use tag selectors */
    h2 {
      font-size: 2rem;
      color: red;
    }

    /* You can use forbidden combinators */
    input[type="text"] + label {
      font-weight: bold;
    }
    
    /* You can style external library classes */
    .slick-dots {
      bottom: 10px;
    }
  }
}
Method 2: Raw Syntax
If you cannot use mixins, use the pseudo-selector directly.
Syntax: [ParentSelector] :iobuilderglobal [GlobalSelector]
Example:
.myHandle :iobuilderglobal .external-class {
  display: block;
}
Note: The builder namespaces the part BEFORE :iobuilderglobal (making it specific to your app version) and leaves the part AFTER it raw.
Method 3: Global Files (Resets)
To apply styles globally without scoping (e.g., resets, typography systems), use specific file naming conventions. The builder treats these files as entirely global.
1. File Naming: Any SCSS file containing the word global in its name.
    ◦ Example: typography.global.scss
2. Directory: Any file located in src/sass/base/resets/.
    ◦ Example: src/sass/base/resets/normalize.scss
WARNING: CSS in these files is injected raw. A generic selector like div { margin: 0; } here will affect every element on the page.
Technical Behavior (Under the Hood)
1. Extraction: The builder scans .scss files for :iobuilderglobal.
2. Transformation: It namespaces the parent selector to the specific app version (e.g., .vtex-store-theme-2-x-wrapperContainer).
3. Transport: The rule is removed from the component's CSS bundle and appended to the end of vtex.login.css.
4. Support: This technique supports media queries (@media) and Tailwind @apply rules.
